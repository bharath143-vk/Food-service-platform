  1)constructor

      Parent constructor
      About.js:15 parent render
      UserClass.js:18 child constructor
      UserClass.js:25 child render
      UserClass.js:22 child mounted
      About.js:12 Parent mounted

    **if two child are there
    Parent constructor
    parent render
    First childchild constructor
    First childchild render
    second childchild constructor
    second childchild render
    <<< Here Dom updation -single batch>>
    First childchild mounted
    second childchild mounted
    Parent mounted

    //after completing child1 render it is not mounting child1 it proceeds to child 2 after child2
    it is mounting (here react is ) ,((updating dom every time is expensive that's why))
    commit phase takes time so it batches all child renders and after it mounts all childs

 ** at the end rather than updating dom for each child each time it will render the batches
 into single batch and updats dom(why like this because updating dom is so expensive)
 here dom performance is optimized
